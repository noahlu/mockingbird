从tab设计看html元素inline和block的区别

2012-09-23 20:00:00 评论

本文的内容分为三个部分：
  1. tab设计的基本原则；
  2. tab设计及编码实践；
  3. 结合设计，探讨HTML元素的inline和block概念。

网页Tab设计基本原则

以前读《见微知著》，里面有专门的篇幅介绍过亚马逊网站的Tab设计，亚马逊的tab设计的如此的出色以至于让后来的电商乃至其他领域的网站们纷纷效仿。其实亚马逊Tab的设计遵循的要点无非就那么几个：1.让当前tab和非当前tab能够明显的区分开；2. 让当前tab和内容能够融合在一起（包括背景颜色和区域之间不要有横线分割）；3. tab数量不宜过多。

下面是css样式

.ui-tab{
background-color: #F5F5F5;
border-bottom: 1px solid #E1E2E2;
height: 38px;
line-height: 38px;*line-height:37px;
}

.ui-tab a{
color: #0077FF;

}

background-color: #ffffff;

border-bottom: #ffffff;

color: #333333;
}

如果你按照这个代码来做的话，很遗憾，在所有浏览器里都不能得到想要的结果。这是因为HTML里的行内元素(inline)在作怪～

HTML元素的inline和block属性

在HTML里，元素有行内(inline)和块级(block)之分，inline元素有a,span,strong等，block元素有p,ul,li,div等，他们的区别之一是block元素总是独占一行，而inline可以从左到右依次排列。

回到上面的问题，因为用作tab的元素是a链接标签，是典型的inline元素，inline元素实际上是不能设置宽高的，它们的宽和高根据里面的内容多少决定，并且inline元素没有上下边距和上下填充。这就导致上面的代码给元素设置margin-bottom:-1px无效。知道原因，解决起来就很简单了，我们可以给元素加上inline-block属性让元素同时具体有inline（左对其）和block（有宽高）的特点：

  1. 增加一条属性.ui-tab a.current{display:inline-block;}
  2. 对ie6再增加一条属性.ui-tab a.current{display:inline-block;zoom:1/*让元素在ie6下haslayout*/}

下面是inline和blcok的区别总结：

  1. 他们的区别你可以简单的理解为inline如同气球，block如同纸盒，你可以在纸盒里放气球，但是不能在气球里放纸盒子。
  2. inline元素不能设置宽高，有左右边距、左右填充，但没有上下边距和上下填充，
  3. 多个block不能并列排列，每个block独占一行。
